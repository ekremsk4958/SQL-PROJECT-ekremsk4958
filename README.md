# SQL Veri Analizi Projesi: [OLİST E-Ticaret Satış Analizi]

## 📊 Proje Hakkında
Bu proje, ham verileri kullanarak işletme kararlarına yön verecek anlamlı içgörüler elde etmek amacıyla hazırlanmıştır. SQL sorguları aracılığıyla gerekli verileri kullanmaya,filtrelemeye ve bu sonuçlar itibari ile çıkarım ve sonuçlara varılmıştır.

## 🛠️ Kullanılan Teknolojiler & Teknikler
* **Dil:** SQL (MS SQL Server/POWER BI/EXCEL )
* **Temel Teknikler:** JOINs, Group By, Having, Order By,Select,Where,datedıff
* **Gelişmiş Teknikler:** Subqueries
* **Veri Temizleme:** COUNT,DISTINCT

## 🚀 Analiz Edilen Sorular
Bu projede şu kritik iş sorularına yanıt aranmıştır:
1. En Çok Ciro Getiren Ürün Kategorileri nelerdir?  
2. Aylık Satış Miktarları nelerdir?
3. Müşterilerin bölge sipariş yoğunlukları nelerdir?
4. Bölgesel sipariş teslimat süreleri nelerdir?
5. Hangi ürün satış adetine oranla daha fazla ciro elde etmektedir?
6. Ödeme yöntemi yoğunluğu hangilerinde daha fazladır?

## 📈 Öne Çıkan Bulgular
  **Pazar Lideri Eyaletler**: Siparişlerin ve toplam cironun %40'ından fazlası yalnızca Sao Paulo (SP) eyaletinde yoğunlaşmaktadır. Bu durum, operasyonel kaynakların ve reklam bütçesinin bu bölgeye odaklanmasının verimliliği artıracağını kanıtlamaktadır.
  
  **Lojistik**: Brezilya geneli ortalama teslimat süresi 293 saat olarak hesaplanmıştır. Bazı uzak eyaletlerde bu sürenin artması, operasyonel iyileştirme fırsatlarını ortaya koymaktadır.

  **Ödeme Sistemleri**: Siparişlerin büyük çoğunluğunda Kredi Kartı tercih edilirken, Brezilya'ya özgü Boleto yönteminin teslimat süresini ortalama 48 saat geciktirdiği gözlemlenmiştir.

  **Bölgesel Dağılım**: Toplam siparişlerin %40'ından fazlası Sao Paulo (SP) eyaletinde yoğunlaşmıştır; bu da lojistik ağının merkezileşmesi gerektiğini göstermektedir.

  **Güneydoğu Dominasyonu**: Satışların büyük bir bölümü Brezilya'nın güneydoğu bölgesinde (SP, RJ, MG eyaletleri) gerçekleşirken, kuzey bölgelerindeki düşük satış hacmi bu pazarlarda büyüme potansiyeli veya lojistik engeller olduğuna işaret etmektedir.
